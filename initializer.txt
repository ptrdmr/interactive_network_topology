# Concourse Bowling — Interactive Network & Device Map

## Project Brief for AI-Assisted IDE

You are building an interactive network infrastructure map for a bowling entertainment venue. The app visualizes device locations on an SVG floor plan with toggleable layers, click-to-inspect device panels, and a searchable device registry. It serves as a single living document replacing scattered spreadsheets and static diagrams.

---

## Tech Stack

### Core
- **Next.js 15** (App Router — `app/`)
- **React 19**
- **TypeScript** (strict mode)
- **Tailwind CSS v4** (`@import "tailwindcss"` in `app/globals.css`, `@tailwindcss/postcss`, `@theme inline` for dark palette and tokens)

### Libraries
- **Recharts** — for any summary/statistics charts (device counts per zone, etc.)
- **Lucide React** — icons throughout the UI
- **@tailwindcss/typography** — prose styling for device detail descriptions

### Data Layer
- All device and floor plan data lives in local JSON files under `data/`
- No database, no API routes — this is a static-data app for now
- Structure the JSON so it could later be swapped for a database or spreadsheet export with zero component changes

## Key Interactions

### Map
- **Pan & zoom** — mouse drag to pan, scroll wheel to zoom, pinch on mobile
- **Device markers** — colored circles/icons based on layer, pulse animation if status is "maintenance"
- **Hover** — tooltip with device name + IP
- **Click** — opens DeviceDetailPanel

### Layer Toggles
- Checkboxes or toggle switches in the sidebar
- Toggling a layer shows/hides all device markers of that type with a fade transition
- "Show All" / "Hide All" buttons at the top
- Layer toggle state persists to localStorage

### Device Detail Panel
- Slides in from right
- Shows all fields from the device JSON
- "Connected to" link that highlights the upstream switch on the map
- Close button + click-outside-to-close

### Device Table View
- Alternate view (tab or route) showing all devices in a sortable, filterable table
- Columns: Name, Type, Zone, IP, Status, Last Serviced
- Click a row to fly to that device on the map

### Search
- Global search bar in the sidebar header
- Searches across: name, IP, MAC, zone, type, notes
- Results shown as DeviceCards in the sidebar
- Click a result to fly to it on the map and open its detail panel

## Floor Plan SVG Strategy

The floor plan is an inline SVG component (`FloorPlanSVG.tsx`). For initial development:

1. **Start with a placeholder rectangle layout** representing zones (server room, lanes, bar/cafe, front desk, office, arcade area)
2. **Each zone is a `<rect>` or `<path>`** with its ID matching `zones.json`
3. **Device markers are rendered as a separate layer ON TOP of the floor plan SVG** (not embedded inside it) — this lets us control z-index and interactions independently
4. **The coordinate system** uses a fixed viewBox (e.g., `0 0 1200 800`) that both the floor plan and device positions reference

Later, the placeholder shapes can be replaced with an actual traced floor plan SVG without touching any other code.

## Notes for the AI IDE

- Do NOT use `pages/` router — this is App Router only (`app/`)
- Tailwind v4 syntax: use `@import "tailwindcss"` not `@tailwind base` directives
- All components are client components where state is needed (`"use client"`)
- Keep the SVG floor plan as a React component, not a static `.svg` file, so zones can respond to hover/click
- The JSON data files in `data/` should be imported directly — no fetch calls needed for static data
- Prefer Lucide React icons over raw SVGs for UI elements
- Use CSS transitions (not framer-motion) to keep the dependency list lean
